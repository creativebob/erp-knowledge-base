# Rules AI: быстрый вход

Этот раздел организован так, чтобы агент читал только нужный минимум:

1. Сначала обязательно читать `rules-ai/project-context.md`.
2. Затем обязательно читать `rules-ai/rules-index.md`.
3. После этого читать только профильные файлы под конкретную задачу.

## Организация файлов правил

- `project-context.md` — инварианты архитектуры и изоляции.
- `rules-index.md` — маршрутизатор: какой rule-файл читать под какой тип задачи.
- `backend/` — правила backend-задач.
- `frontend/` — правила frontend-задач.

## Обязательное правило поддержки

Любые изменения правил в `rules-ai/` вносить строго по этой организации:

- не смешивать backend и frontend правила в одном файле;
- не дублировать обязательные пункты между файлами;
- общие инварианты хранить только в `project-context.md`;
- маршрутизацию чтения держать только в `rules-index.md`;
- длинные примеры и справочные материалы хранить в профильных файлах и ссылаться на них из index/контекстных файлов.
